# Core

Use `core` para codigo que existe uma vez na aplicacao inteira.

Coloque aqui:

- providers globais;
- configuracoes da aplicacao;
- interceptors HTTP;
- guards globais;
- services singleton;
- integracoes com APIs, storage, analytics ou autenticacao.

Evite colocar componentes de tela ou codigo especifico de uma feature. Se algo pertence a uma pagina ou dominio, deixe em `features`.
